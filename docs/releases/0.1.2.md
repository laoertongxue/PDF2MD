# PDF2MD 0.1.2

This release delivers the first macOS Apple Silicon desktop package for the MBA intensive-reading
workbench.

## Highlights

- Manage multiple PDF textbooks inside one course and keep chapters tied to their source material.
- Run an unattended OCR chain using Apple Vision, Codex CLI, conditional Baidu OCR escalation, and
  Codex final adjudication.
- Generate evidence-bound Markdown notes with concept explanations, cases, practical problem
  solving, applications, and directly previewable Mermaid diagrams.
- Use the fixed DeepSeek model `deepseek-v4-pro`; store the API key in macOS Keychain.
- Download `PDF2MD_<version>_aarch64.dmg` or the equivalent `.app.zip`, with matching SHA-256 files.

## Release Verification

The release workflow builds on an arm64 `macos-14` runner and verifies the Apple Silicon app,
bundled Python runtime, OCR schemas, sidecar path safety, restricted-environment `/health` startup,
and ad-hoc signature before publishing assets. GitHub artifact provenance is attested.

## Important Limitations

- The package is ad-hoc signed and not Apple-notarized. macOS may require the Finder **Open** or
  System Settings **Open Anyway** flow on first launch.
- Codex CLI must be a secure directly executable file. A symlink is intentionally rejected.
- Baidu OCR is optional and is used only for conflict, complex, or deterministic sample pages;
  missing configuration blocks those pages rather than silently lowering quality.
- Real scanned textbook checks have validated Apple Vision rendering and caching, but the current
  validation machine's Codex CLI is a symlink. Full unattended textbook publication is therefore
  not claimed until a secure Codex CLI executable is configured.
